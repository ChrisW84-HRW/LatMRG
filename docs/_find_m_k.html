A software package to test and search for new linear congruential random number generators
FindMK

This module contains a program to search for good modulus for MRGs.

Given a specific order, this program will search for modulus that are of one of two forms :

  • Modulus that are smaller than a particular power of 2
  • Modulus that are in a specific interval

In the first case, the program will search for \(m\) prime such that \(m < 2^e\) and for which \(r = (m^k-1)/(m-1)\) is also prime. We may also require that \((m-1)/2\) be prime.

In the second case, the program will search for all prime integers in a given interval such that \( 2^e + c_1 \leq m \leq 2^e + c2\). It is also possible to require to have \((m-1)/2\) be prime.

In both cases, it is possible to ask to program to factorize \(m-1\). This factorization will then be printed in a file with the extension .fac. This program uses the class LatMRG::Primes to perform its computations, and this class uses LatticeTester::IntFactor.

This program is called FindMK. The program reads the search parameters from a file. This data file must have the extension .dat. To run the program, a combination of types and the name of the data file must be given without extension on the command line, and the results will be written in a file of the same name but with extension .res. For example, if the data file is called find.dat, then the program is ran by calling 

 FindMK <I,Z> find

and the results will be written in file find.res.

The data file must have the following format: 

 power
 k
 e
 c1 [c2]
 safe
 facto
Data file format for FindMK

The file must contain the following parameters in that order:

power :
<bool> if power is true, the program will search for prime integers smaller than \(2^e\), otherwise the program will search in an interval.
k :
<int> the order of the MRG that you search an m for. Note that k has to be odd because there are no prime r for k even.
e :
<int> the e for which the program will search around \(2^e\).
c1 [c2] :
<int> <int> if power is false, the program will use \(c_1\) and \(c_2\) limit the research interval. If power is true, the program will make computations until it has found \(c_1\) primes meeting the constraints.
safe :
<bool> if true, only \(m\) such that \((m-1)/2\) is prime are considered.
facto :
<bool> if true, the program will factorize \(m-1\). It is recommended to set this option to false, especially if safe is true.
